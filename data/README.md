# Enterprise SaaS Dataset Documentation & Data Architecture

## Overview

This repository uses a synthetic enterprise SaaS dataset designed to simulate the operational data generated by a mid-sized subscription business.

The dataset supports multiple business functions including:

- Revenue Intelligence
- Customer Health Monitoring
- Subscription Analytics
- Revenue Operations
- Product Intelligence
- Customer Retention Analytics
- Billing & Collection Monitoring
- Executive-Level Business Reporting

Rather than supporting a single analytical use case, the dataset was intentionally designed to answer a broad range of business questions across Finance, Product, Revenue Operations, Customer Success, and Leadership teams.

Examples include:

> - Is Monthly Recurring Revenue growing sustainably?
> - Which customers are most likely to churn?
> - How much revenue remains uncollected?
> - Which subscription plans drive business growth?
> - Which customers are disengaging from the product?
> - Where is revenue leakage occurring?
> - Which accounts require proactive intervention?

The dataset provides the analytical foundation for all three projects within this repository.

## Subscriptions (5,000 Records)

| Column | Type | Description |
|--------|------|------------|
| SubscriptionID | Integer | Primary Key |
| CustomerID | Integer | Foreign Key → Customers |
| PlanID | Integer | Foreign Key → Subscription Plans |
| StartDate | Date | Subscription start date |
| EndDate | Date | Subscription end date |
| Status | Text | Current subscription status |

### Subscription Status Distribution

| Status | Count |
|-------|------|
| Active | 1,776 |
| Past Due | 1,772 |
| Cancelled | 1,452 |

### Business Insight

Perhaps the most significant characteristic of this dataset is that the number of Past Due subscriptions is nearly identical to the number of Active subscriptions.

This creates realistic business scenarios involving:

- Subscription Health Monitoring
- Revenue Leakage Detection
- Customer Retention Analysis
- Churn Prevention Strategies

Past Due subscriptions serve as leading indicators of potential customer churn throughout multiple projects within this repository.

## Data Quality & Validation

All datasets were validated prior to project development using the validation procedures implemented throughout the repository.

Validation checks included:

- Missing Customer IDs
- Missing Usage Dates
- Negative Invoice Values
- Negative Payment Values
- Subscription Integrity Checks
- Invoice-Payment Relationship Validation
- Duplicate Detection Procedures

### Validation Results

The current dataset contains:

> - No negative invoice amounts.
> - No negative payment amounts.
> - No missing Customer IDs.
> - No missing Usage Dates.

Although the current dataset validates successfully, the cleaning and validation procedures remain intentionally included throughout each project.

Real-world analytical environments are rarely perfect, and defensive data validation practices are critical for maintaining reliable business reporting at scale.
